# GardenSuite Landing Page - Master Direction

This is the current source of truth for the GardenSuite homepage direction.

It replaces the earlier mix of:

- inspiration notes
- reference extraction docs
- one-off design audits
- copy review notes

## What We Are Building

GardenSuite is not a generic SaaS tool. It is a tea garden ERP with a very specific operational promise:

- face attendance that verifies the worker
- smart wireless weighing linked to the worker record
- payroll based on connected field data
- daily MIS visibility for management
- offline-first reliability at the garden

The landing page should feel clear, serious, and established. It should not feel like a startup template, a scenic brand film, or a generic software site.

## Final Direction

- Light mode only
- Keep the current hero's large-scale impact
- Keep the current hero font direction
- Keep the GardenSuite and tea-garden identity
- Rebuild the rest of the page so it matches the hero's confidence

## Current Diagnosis

The homepage problem is not only copy. It is a coherence problem.

### What currently works

- The hero has impact
- The hero typography feels strong enough to keep
- The product category is valuable and differentiated
- The named estate proof is real and worth surfacing
- Sarbani Associates is a trust advantage

### What currently feels off

- The hero feels branded and cinematic, but the body feels assembled from SaaS patterns
- Too many sections overlap in purpose
- Too many visuals are synthetic instead of product-truthful
- Too many sections reuse the same soft card treatment
- The page moves between scenic, editorial, and utility UI languages without choosing one

## What To Learn From The Inspiration Sites

We are not copying one site. We are borrowing discipline from the best parts of several.

### Primary reference behavior

- Vercel - strong hero clarity, hard layout discipline, less decorative softness
- Clerk - clear product category framing, light-mode authority, structured trust
- Fey - generous whitespace, product-first presentation, editorial calm
- Modeinspect - minimal layout, strong large-scale proof sections
- Keytail - typography restraint, spacing discipline, clean rhythm

### What not to copy literally

- Linear and Raycast dark-mode atmosphere
- abstract startup metaphors
- decorative dark gradients
- generic enterprise jargon
- product-agnostic feature grids

## Core Principles

1. Category before slogan
2. Proof before flourish
3. Operational truth before metaphor
4. Fewer sections, each with a clear job
5. One light-mode visual system across the whole page
6. Screenshots and real workflow proof should carry authority

## Page Structure To Build Toward

### 1. Hero

Keep:

- current large-scale visual impact
- current headline/subtitle font direction
- light scenic identity only if it supports the product

Change:

- simplify scenic layering
- ensure product proof remains the main focal point
- make the supporting copy more category-led and operational

### 2. Problem Strip

Name the pain the audience recognizes:

- registers getting lost
- buddy punching and inflated weights
- payroll taking days

This earns emotional buy-in before showing the solution. Three columns, short and visceral.

### 3. Product Grid

Show all 5 modules as cards linking to their product pages:

- Face Attendance & Smart Weighing (featured, larger card)
- Automated Payroll
- Factory Production
- Store Management
- Daily MIS Report

Each card should include a mini UI preview (CSS-recreated from real app screenshots when available), one sharp headline, and a one-liner description. The section heading should frame the connected system, not just list features.

This replaces the earlier connected workflow + module detail sections.

### 4. Proof Strip

Social proof repositioned after products:

- 20+ tea estates
- 7 regions
- 25+ years
- built and supported by Sarbani Associates

Placed after products so the visitor has seen the solution before being asked to trust the company. Proof answers "can I trust these people?" at the moment it matters.

### 5. Differentiator

Three cards explaining why GardenSuite over generic ERPs:

- tea garden workflow built in
- runs offline at the garden
- setup and support on site by Sarbani Associates

No stats row here (stats are in the proof strip above).

### 6. FAQ

Keep only genuine objections:

- works without internet
- setup time
- region coverage
- owner visibility outside the garden
- pricing model

### 7. CTA

Short, direct, low-drama CTA. No extra aesthetic trick needed.

## Section Design Rules

- Alternate section backgrounds between white and off-white to create rhythm
- Problem strip and proof strip should feel tight and factual
- Product grid cards should feel interactive and route to product pages
- Differentiator cards should feel calm and authoritative
- Let FAQ and CTA sections breathe with plain backgrounds
- Use accent color sparingly and intentionally

## Visual System Rules

- Light mode only
- White should be the default background
- Warm off-white should be a supporting background, not the whole page
- Green is an accent, not a page wash
- Reduce decorative glows
- Reduce giant radii on repeated UI blocks
- Use real screenshots wherever possible
- If an illustration is used, it must explain a real garden workflow, not simulate software polish

## Typography Decision

Do not change the hero font direction unless later visual testing proves it is the actual problem.

Current view:

- hero title and subtitle typography are already one of the stronger parts of the page
- typography should be tightened and unified below the hero
- the body of the page should rise to the hero, not drag the hero down

See `typography.md`.

## Copy Decision

The page should sound like someone who understands tea garden operations, not like a SaaS copywriter describing software categories in the abstract.

Use:

- hazira
- plucking weight
- payroll
- stores
- factory
- daily MIS
- estate office
- offline at the garden

Avoid:

- empty slogans
- generic efficiency language
- overclaiming without visible proof

See `copy.md`.

## What We Keep vs What We Change

### Keep

- hero impact
- hero typography direction
- light-mode direction
- tea-garden brand identity
- Sarbani Associates trust positioning

### Change

- sections after the hero
- copy hierarchy
- repeated card styling
- decorative scenic weight
- fake operational visuals
- overall section rhythm

## Execution Priority

1. Restructure the page sections
2. Replace synthetic visuals with real product proof
3. Rewrite section copy to match the new structure
4. Tighten spacing, radii, and background usage
5. Revisit motion only after the static page feels coherent
