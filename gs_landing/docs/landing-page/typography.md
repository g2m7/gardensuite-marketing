# GardenSuite Landing Page - Typography Direction

## Decision

Keep the current hero typography direction unless the redesign proves it is the actual source of inconsistency.

Current judgement:

- the hero title has enough presence
- the hero subtitle feels calm and readable
- the stronger typography problem is below the hero, where the hierarchy becomes too safe and repetitive

## Overall Type Strategy

- one clear display voice
- one supporting text voice
- tighter hierarchy below the hero
- less variation, more discipline

## Reference Lessons

From Keytail, Vercel, Clerk, Fey, and Modeinspect:

- large headings feel premium when spacing is disciplined
- negative tracking on display text adds authority when used carefully
- body text stays neutral and readable
- headings work best when there are fewer intermediate sizes

## Type System Priorities

### 1. Hero

Keep:

- current display feel
- strong size contrast
- balanced subtitle line length

### 2. Section Headings

Need:

- fewer sizes
- tighter rhythm
- stronger distinction between section headings and card headings

### 3. Card / Module Headings

Need:

- smaller, calmer headings
- less visual competition with section headings

### 4. Body Text

Need:

- simple readable rhythm
- slightly narrower max widths
- consistent muted color

## Recommended Rules

- Use one font family if possible across the page
- Keep hero display styling as the premium top of the system
- Use negative tracking only on display and major section headings
- Use positive tracking only on tiny uppercase labels
- Keep body copy in a narrow range of sizes

## Directional Scale

This is a direction, not a locked token file:

- Hero H1: very large, tight line-height, slightly negative tracking
- Hero subtitle: large body size, comfortable line-height
- Section H2: clearly smaller than hero, still confident
- Module H3: compact and factual
- Body: 16px range
- Labels: 12px to 13px uppercase only where structurally useful

## Problems To Avoid

- too many mid-sized headings
- oversized headings inside compact cards
- card titles that visually compete with section titles
- mixing editorial type mood and product UI type mood without hierarchy

## Practical Outcome

Typography should support this hierarchy:

1. Hero promise
2. Section argument
3. Product/module explanation
4. Supporting proof

Right now the hero mostly works. The rest of the page needs to align to it.
